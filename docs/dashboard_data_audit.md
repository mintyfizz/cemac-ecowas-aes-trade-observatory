# Dashboard Data Audit

Generated: 2026-05-22

Status: **PENDING LIVE VALIDATION**

This report documents the expected data contract for the GitHub Pages static
dashboard. The committed dashboard shell reads only local files from
`static/data/*.json`; Databricks gold tables remain the source of truth and are
exported by `scripts/export_static.py` during the Pages workflow.

Local validation note: the local `.env` available during this implementation
does not contain `DATABRICKS_HOST`, `DATABRICKS_HTTP_PATH`, or
`DATABRICKS_TOKEN`, and `static/data/` is generated by CI rather than committed.
Therefore, the repository can be syntax-checked locally, but row-count and
formula validation must run after Databricks credentials are available.

## Formula Checks

`scripts/audit_dashboard_data.py` validates these formulas where the required
source fields are present:

- `total_trade = exports + imports`
- `trade_balance = exports - imports`
- `trade_openness_pct_gdp = total_trade / GDP * 100`
- `exports_pct_gdp = exports / GDP * 100`
- `imports_pct_gdp = imports / GDP * 100`
- `gdp_per_capita = GDP billions / population millions * 1000`
- `total_trade_partner_hhi` stays within `[0, 1]`

Missing inputs must remain `null`. The frontend should render `--`, an empty
chart point, or a clear no-coverage state rather than turning missing values
into zero.

## Panel Interpretation Notes

| Panel | Interpretation rule |
| --- | --- |
| Overview cards | Summarize only the selected bloc/country and selected year. YoY badges compare to the previous calendar year when available. |
| Map | Uses selected-year values from `country_timeseries.json`. Clicking a country switches the dashboard to country-only mode. |
| Top trading partners | Uses country partners only; aggregate partner rows are filtered out of displayed outputs. |
| Partner share comparison | Partner share is denominated by full total trade, not only top-10 partners. |
| Trade growth indexed to 1990 | Uses nominal total goods trade from loaded IMTS/DOTS data: `(exports + imports) / 1990 total trade * 100`. Missing years remain gaps. |
| Trade exposure profile | Shows exports, imports, and trade balance as shares of GDP. |
| Product trade structure | Uses selected-year HS2 product data only. If selected-year coverage is missing, the dashboard shows latest available product year instead of inventing values. |
| Conflict | Bloc view ranks member countries; country drilldown ranks admin1 regions. |
| Fragility | FSI is latest-available coverage. If no 2024 FSI source release exists, the dashboard must not imply true 2024 FSI data. |
| Macro-fiscal pressure | A normalized dashboard diagnostic, not a sovereign rating, credit rating, or debt sustainability assessment. |

## Known Gaps

- FSI coverage is latest available and should currently be treated as
  2006-2023 unless the source export proves a newer release.
- ACLED loaded coverage starts in 1997 even though the dashboard year selector
  starts in 1990.
- `gold.product_trade_hs2` coverage depends on reporter submissions. Some
  selected years may have no HS2 product data for a country or bloc.
- Mirror gaps and product-level HS6 composition are not displayed unless a
  properly loaded source table supports them.

## Validation Command

Run this after exporting static JSON:

```bash
python scripts/export_static.py
python scripts/audit_dashboard_data.py --report docs/dashboard_data_audit.md
```

For CI, the workflow runs:

```bash
python scripts/audit_dashboard_data.py --check-only
```

The CI check fails when required JSON files are missing, row-count contracts are
broken, formulas fail, or live Databricks validation cannot run.
